## 🔄 Phase 1: Data Generation

### Dataset Creation
**Script**: [generate_dataset.py](file:///d:/Deepfocus/rgbd_focalstack_loss/generate_dataset.py)

**Process**:
1. Run DeepFocus LFS inference on seed0000-seed0129 (130 scenes)
2. Generate 40 focal planes per scene (diopter 0.1 to 4.0)
3. Compute quality metrics against ground truth:
   - MSE (Mean Squared Error) → Later converted to PSNR
   - SSIM (Structural Similarity)
   - LPIPS (Learned Perceptual Image Patch Similarity)
4. Save predictions as `.exr` and labels as `labels.json`

**Dataset Split**:
- **Training**: seed0000-seed0089 (90 scenes × 40 planes = 3600 samples)
- **Validation**: seed0090-seed0109 (20 scenes × 40 planes = 800 samples)  
- **Test**: seed0110-seed0129 (20 scenes × 40 planes = 800 samples)

**Key Finding**: Initial MSE values were too small (~ 1e-5), causing learning instability. **Solution**: Convert to PSNR for better dynamic range.

## 🏗️ Phase 2: Model Implementation

### Network Architecture
**File**: [model.py](file:///d:/Deepfocus/rgbd_focalstack_loss/model.py)

**LossEstimationNet**:
```python
Input: (8, 512, 512)
  ├─ RGBD: 4 channels (RGB + Depth/12.0)
  ├─ Predicted focal plane: 3 channels (RGB)
  └─ Diopter: 1 spatial channel (added in forward())

Architecture:
  ├─ Custom space_to_depth (TF-compatible, block_size=2) → (32, 256, 256)
  ├─ ResNet-like backbone with 4 blocks
  │   ├─ Block1: 32 → 64 channels
  │   ├─ Block2: 64 → 128 channels
  │   ├─ Block3: 128 → 256 channels
  │   └─ Block4: 256 → 512 channels
  ├─ Global Average Pooling → (512,)
  └─ Multi-head output:
      ├─ fc_psnr → ReLU → PSNR (dB)
      ├─ fc_ssim → Sigmoid → SSIM [0,1]
      └─ fc_lpips → Sigmoid → LPIPS [0,1]

Parameters: ~33M
```

**Key Design Choices**:
- TF-compatible `space_to_depth`: Channel ordering `(r_h, r_w, c)` instead of PyTorch's `(c, r_h, r_w)`
- PSNR output: ReLU activation (non-negative)
- SSIM/LPIPS: Sigmoid activation (bounded to [0,1])

### Dataset Loading
**File**: [dataset.py](file:///d:/Deepfocus/rgbd_focalstack_loss/dataset.py)

**Preprocessing**:
```python
RGB normalization:
  rgb = rgb^(1/2.2)  # Gamma correction
  rgb = rgb / (max(rgb) + 1e-8)

Depth normalization:
  depth = depth / 12.0

MSE → PSNR conversion:
  psnr = -10 * log10(mse + 1e-10) if mse > 0 else 100.0
```

### Training Script
**File**: [train.py](file:///d:/Deepfocus/rgbd_focalstack_loss/train.py)

**Configuration**:
- Optimizer: Adam (lr=1e-4, weight_decay=1e-5)
- Loss: L1Loss (MAE) for all three metrics
- Multi-task loss: `loss = psnr_loss + ssim_loss + lpips_loss` (equal weights)
- Scheduler: ReduceLROnPlateau (factor=0.5, patience=5)
- Batch size: 8
- Epochs: 50

## 🔍 Key Findings & Issues

### ✅ What Worked Well

1. **MSE → PSNR Conversion**
   - Original MSE values (~1e-5) caused gradient issues
   - PSNR (40-50 dB range) provided stable learning
   - Much better interpretability

2. **SSIM Prediction**
   - Best-performing metric (0.79 Pearson correlation)
   - Low relative error (0.2%)
   - Reliable enough for quality assessment

3. **Architecture Design**
   - ResNet backbone effective for feature extraction
   - Multi-task learning works reasonably well
   - TF-compatible operations successful

### ⚠️ Identified Problems

1. **LPIPS Saturation** (Critical)
   - Model cannot predict poor quality (high LPIPS)
   - All training data has low LPIPS → network learns to predict narrow range
   - 24% relative error unacceptable

2. **Dataset Bias**
   - DeepFocus LFS always produces high-quality focal stacks
   - No "bad quality" examples in training set
   - Model never learns what poor predictions look like

3. **Early Convergence**
   - Best model at epoch 3 suggests:
     - Limited diversity in data
     - Or overfitting to high-quality samples
     - May need regularization or data augmentation

4. **PSNR Wavelike Pattern**
   - Suggests systematic per-diopter bias
   - Some focal distances predicted better than others
   - May need diopter-aware architecture or normalization

## 🔄 Phase 4: Data Augmentation Implementation

### Strategy
To address **LPIPS saturation** (model only predicting high quality), we implemented a comprehensive data augmentation pipeline:

1. **Strong Augmentations** (Target PSNR: 15-35 dB)
   - Heavy Gaussian/Poisson noise
   - Strong Gaussian blur (sigma 0.5-3.0)
   - JPEG compression (quality 30-80)
   - Chromatic aberration
   - **Goal**: Force model to learn "bad quality" features

2. **Weak Augmentations** (Target PSNR: 30-40 dB)
   - Mild noise/blur to bridge the gap between Clean and Strong
   - **Goal**: Improve fine-grained quality discrimination

3. **All-in-Focus (AiF)**
   - Use ground truth `clean_pass_rgb` as the "perfect" focal plane prediction
   - **Goal**: Provide anchor points for perfect quality (LPIPS ≈ 0)

### New Scripts
**1. [`augmentations.py`](file:///d:/Deepfocus/rgbd_focalstack_loss/augmentations.py)**
- Core augmentation logic using `numpy`, `scipy`, `PIL`
- No OpenCV dependency for easier deployment

**2. [`generate_augmented_dataset.py`](file:///d:/Deepfocus/rgbd_focalstack_loss/generate_augmented_dataset.py)**
- Generates augmented data with quality control
- Retries augmentation if LPIPS is out of target range
- Creates:
  - `aug_pred_frame_*.exr` + `aug_labels.json`
  - `weak_pred_frame_*.exr` + `weak_labels.json`
  - `aif_labels.json` (uses existing clean RGB)

### Updated Training Pipeline
- **Dataset**: `dataset.py` now supports mixing Clean, Strong, Weak, and AiF data
- **Loss**: `train.py` supports weighted loss (e.g., `lpips=2.0`) to prioritize perceptual metrics

## 🛡️ Phase 5: Verification & Robustness

### 1. Adversarial Training (train.py)
**Objective**: Prevent the model from being fooled by "fake good" images (images optimized to high predicted scores but low perceptual quality).

**Mechanism**:
- Every `N` epochs, generate **adversarial samples**:
  1. Start with clean RGB + noise
  2. Optimize pixels to *minimize* the model's predicted loss (cheat the metric)
  3. Compute **REAL** metrics (PSNR/SSIM/LPIPS) for these "fake" images
- **Train** the model on these samples using their real (poor) metrics as targets.
- **Result**: Model learns that these specific artifacts correspond to bad quality, not good quality.

### 2. Loss Function Verification (`verify_as_loss.py`)
**Objective**: Prove that the trained model provides valid gradients for image optimization.

**Process**:
1. Freeze the trained Loss Estimation Model.
2. Start with a noisy image.
3. Optimize image pixels using gradients from the model (Gradient Descent).
4. **Success criteria**: If the optimized image actually gets closer to Ground Truth (Real PSNR↑, Real SSIM↑, Real LPIPS↓), the model is a valid differentiable loss function.
5. Generates `convergence.png` and `comparison.png` to visualize the optimization trajectory.
